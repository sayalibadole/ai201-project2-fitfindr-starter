### Tool 1: search_listings

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
Searches the secondhand listings dataset to find items that match the user's requested item description, size, and budget. Results are filtered and ranked by relevance so the agent can select the best match.

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- description (str): Keywords describing the item the user is looking for (e.g., "vintage graphic tee", "y2k - denim jacket").
- size (str): Desired clothing size (e.g., XS, S, M, L).
- max_price (float): Maximum amount the user is willing to spend.

**What it returns:**
<!-- Describe the return value — what fields does a result contain? -->
A list of matching listings, where each listing contains:

id
title
description
category
style_tags
size
condition
price
colors
brand
platform

The results are sorted by relevance, and the agent typically selects the top match for the next step.

**What happens if it fails or returns nothing:**
<!-- What should the agent do if no listings match? -->
If no listings match, the tool returns an empty result and a message explaining why. The agent informs the user and either:

Retries with relaxed constraints (e.g., ignores size requirements), or
Asks the user to broaden their search criteria, such as increasing their budget or changing keywords.

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
Analyzes the selected thrifted item and the user's wardrobe to generate one or more complete outfit combinations that match the item's style.

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- new_item (dict): The selected listing returned by search_listings.
- wardrobe (list[dict]): The user's existing wardrobe items.

**What it returns:**
<!-- Describe the return value -->
One or more outfit recommendations containing:

Suggested wardrobe pieces
Styling tips
Overall aesthetic/style description
Explanation of why the pieces work together

**What happens if it fails or returns nothing:**
<!-- What should the agent do if the wardrobe is empty or no outfit can be suggested? -->
If the wardrobe is empty or lacks compatible pieces, the tool generates a simplified outfit suggestion using only the new item and general styling advice. The agent explains that wardrobe-based matching was limited due to insufficient items.

## Planning Loop

**How does your agent decide which tool to call next?**
<!-- Describe the logic your planning loop uses. What does it look at? What conditions change its behavior? How does it know when it's done? -->

1) The agent begins by collecting the user's search criteria (description, size, and max_price) and calls search_listings().
2) After search_listings() returns:
     - If the results list is empty:
          - Store an error message in the session state.
          - Inform the user that no matching items were found.
          - Optionally retry with relaxed constraints (for example, removing the size filter).
          - If the retry also returns no results, ask the user to modify their search and end the workflow.
     - If results are found:
          - Set selected_item = results[0] (the highest-ranked result).
          - Store selected_item in session state.
          - Proceed to suggest_outfit().
3) The agent calls suggest_outfit(selected_item, wardrobe).
4) After suggest_outfit() returns:
     - If a valid outfit suggestion is generated:
          - Store the outfit in session state as selected_outfit.
          - Proceed to create_fit_card().
     - If the wardrobe is empty or no matching pieces are found:
          - Generate a fallback outfit recommendation using only the new item and general styling advice.
          - Store the fallback outfit as selected_outfit.
          - Proceed to create_fit_card().
5) The agent calls create_fit_card(selected_outfit, selected_item).
6) After create_fit_card() returns:
     - If a fit card is successfully generated:
          - Display the selected item, outfit recommendation, and fit card to the user.
          - Mark the workflow as complete.
     - If fit card generation fails:
          - Generate a simple template-based caption using the item's information.
          - Display the fallback caption.
          - Mark the workflow as complete.

The agent knows it is finished when a fit card (either generated or fallback) has been returned to the user, or when the search step fails and no valid listings can be found after retrying.

## State Management

**How does information from one tool get passed to the next?**
<!-- Describe how your agent stores and accesses state within a session. What data is tracked? How is it passed between tool calls? -->

The agent maintains a single session dictionary that persists across the entire interaction and acts as the shared memory between all tool calls. Each step of the planning loop reads from and writes to this session object, ensuring that outputs from earlier tools are available to later tools without requiring the user to repeat information.

At initialization, the session stores:

query: original user request
parsed: structured version of the query (description, size, max_price)
search_results: output of search_listings
selected_item: the single listing chosen from search results
outfit_suggestion: output of suggest_outfit
fit_card: final generated caption from create_fit_card
error: any failure message that stops execution early

Data flows strictly forward through the session:

After parsing the query, parsed is stored in the session and used as input to search_listings.
The output of search_listings is stored in search_results. If it is non-empty, the first item is saved as selected_item.
selected_item is passed into suggest_outfit, and the returned outfit is stored in outfit_suggestion.
outfit_suggestion and selected_item are then passed into create_fit_card, and the result is stored in fit_card.

The agent does not recompute earlier steps once they are stored; instead, it always reads from the session state. If an error occurs at any stage (most commonly in search_listings), the error field is set and the function returns early, preventing downstream tools from being called with invalid or missing data.

The workflow is considered complete when either:

fit_card is successfully generated and stored in the session, or
error is set due to an unrecoverable failure (e.g., no search results even after retry logic).

## Error Handling

For each tool, describe the specific failure mode you're handling and what the agent does in response.

| Tool | Failure mode | Agent response |
|------|-------------|----------------|
| search_listings | No results match the query | The agent immediately stops the workflow and tells the user: “I couldn’t find any vintage graphic tees under $30 in your size. Try widening your search (e.g., increase budget to $40, remove size constraint, or try related terms like ‘retro graphic tee’ or ‘band tee’).” The agent does not call any downstream tools. Optionally, it may retry once with relaxed constraints (e.g., drop size filter) and clearly state what was changed before retrying. |
| suggest_outfit | Wardrobe is empty | The agent still generates an outfit using only the selected item. It responds with something like: “I don’t have any wardrobe items to mix with this, so here’s a standalone styling idea: pair the tee with baggy jeans and chunky sneakers for a clean 90s streetwear look, and add a flannel or hoodie for layering.” It explicitly notes that wardrobe-based personalization was unavailable but continues the workflow instead of failing. |
| create_fit_card | Outfit input is missing or incomplete | The agent reconstructs a fallback using selected_item and any partial outfit text. It generates a simple caption such as: “thrifted this $22 vintage graphic tee and styled it into a baggy-jeans streetwear fit 🖤” It avoids crashing or returning empty output and instead produces a minimal but usable social caption so the user still receives a complete final result. |

## A Complete Interaction (Step by Step)

FitFindr is an AI-powered thrifting assistant that helps users find secondhand clothing items, style them with pieces they already own, and generate a shareable outfit caption. The agent uses a planning loop to decide which tool to call next: it first searches for matching listings, then suggests outfits using the selected item and the user's wardrobe, and finally creates a fit card based on the completed outfit. If any tool fails or returns no useful results, the agent informs the user, applies a fallback strategy when possible (such as relaxing search filters), or requests additional information instead of continuing with invalid data.
